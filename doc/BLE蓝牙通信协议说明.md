## 2. 综述

该协议规定了BMS电池管理系统与手机蓝牙之间数据交互的流程、格式和内容。通过蓝牙连接，手机端可以读取电池的基本信息和设置电池的保护参数。

---

## 3. 通信规约

BMS和手机蓝牙采用主从机制，一问一答，手机为主，BMS为从。主机读取时直接发送对应的功能码，不带数据块。

### 3.1 BLE GATT 与编码

| 项目 | 说明 |
|------|------|
| **接收（手机侧收数）** | 使用 **FFF1** 服务/特征（UUID 短形式 `0xFFF1`）。手机端须对该特征 **启用 Notify（通知）** 后，方可接收从机上报的协议帧。 |
| **发送（手机侧发数）** | 使用 **FFF2** 服务/特征（UUID 短形式 `0xFFF2`）。主机通过 **Write** 向从机下发指令帧。 |
| **字符编码** | 协议中涉及文本的字段统一采用 **GB18030** 编码（与 ASCII 兼容的字符编码结果与 ASCII 一致）。 |

> **说明：** 不同蓝牙模块对「服务」与「特征」的文档表述可能不同；若模块将 `0xFFF1` / `0xFFF2` 挂在同一自定义服务下，请以模块厂商说明为准，本协议以 **接收走 FFF1 + Notify、发送走 FFF2** 为准。

## 4. 主协议格式

| 起始位（1 Byte） | 功能码（2 Byte） | 数据长度（2 Byte） | 数据块（N Byte） | XOR校验值（2 Byte） | 结束位（2 Byte） |
|:-:|:-:|:-:|:-:|:-:|:-:|
| 帧头 0x55 | 功能码用于区分当前指令功能 | 数据块的长度 | 数据内容 | 对起始位、功能码、数据长度、数据块所有字节进行异或操作 | 帧尾 0xAABB |

**说明：**
- 多字节字段均采用**大端**字节序（高字节在前）
- XOR校验范围：起始位 + 功能码 + 数据长度 + 数据块，逐字节异或
- 数据长度为数据块的实际字节数

## 6. 协议信息

### 功能码 - 查询电池信息：0x1001

### 功能码 - 设置保护参数：0x1002

用于设置BMS的保护参数，支持设置一级放电过流保护值和充电过流保护值。

#### 主机发送

| 字段 | 长度 | 内容 |
|------|:----:|------|
| 设置参数类型 | 1 Byte | 0x00: 一级放电过流保护值<br>0x01: 充电过流保护值 |
| 设置参数值 | 2 Byte | 小端，Unit: 0.1A |

**参数类型与范围说明：**

| 参数类型 | 描述 | 单位 | 范围说明 |
|:--------:|------|:----:|----------|
| 0x00 | 一级放电过流保护值 | 0.1A | 不低于放电过流保护下限，不高于放电过流保护上限 |
| 0x01 | 充电过流保护值 | 0.1A | 不低于充电过流保护下限，不高于充电过流保护上限 |

#### 从机回复

| 字段 | 长度 | 内容 |
|------|:----:|------|
| 设置结果 | 1 Byte | 0x00: 设置失败，请确认参数范围<br>0x01: 设置成功 |

## 7. 通信示例

### 示例1：查询电池信息

**主机发送（查询命令）：**

```
55 10 01 00 00 [XOR_H] [XOR_L] AA BB
```

| 字节 | 说明 |
|------|------|
| 0x55 | 帧头 |
| 0x10 0x01 | 功能码 0x1001（查询电池信息） |
| 0x00 0x00 | 数据长度 = 0 |
| [XOR] | 2字节XOR校验 |
| 0xAA 0xBB | 帧尾 |

**从机回复：**

```
55 10 01 [LEN_H] [LEN_L] [数据块...] [XOR_H] [XOR_L] AA BB
```

### 示例2：设置一级放电过流保护值为60A

**主机发送：**

```
55 10 02 00 03 00 58 02 [XOR_H] [XOR_L] AA BB
```

| 字节 | 说明 |
|------|------|
| 0x55 | 帧头 |
| 0x10 0x02 | 功能码 0x1002（设置保护参数） |
| 0x00 0x03 | 数据长度 = 3 |
| 0x00 | 参数类型 = 0x00（一级放电过流保护值） |
| 0x58 0x02 | 参数值 = 0x0258 = 600（小端），即 600 × 0.1A = 60A |
| [XOR] | 2字节XOR校验 |
| 0xAA 0xBB | 帧尾 |

**从机回复（设置成功）：**

```
55 10 02 00 01 01 [XOR_H] [XOR_L] AA BB
```

### 示例3：设置充电过流保护值为15A

**主机发送：**

```
55 10 02 00 03 01 96 00 [XOR_H] [XOR_L] AA BB
```

| 字节 | 说明 |
|------|------|
| 0x55 | 帧头 |
| 0x10 0x02 | 功能码 0x1002（设置保护参数） |
| 0x00 0x03 | 数据长度 = 3 |
| 0x01 | 参数类型 = 0x01（充电过流保护值） |
| 0x96 0x00 | 参数值 = 0x0096 = 150（小端），即 150 × 0.1A = 15A |
| [XOR] | 2字节XOR校验 |
| 0xAA 0xBB | 帧尾 |

## 8. 注意事项

1. **主从机制**：手机为主机，BMS为从机，所有通信由手机发起
2. **一问一答**：主机发送一条指令后，需等待从机回复后再发送下一条指令
3. **XOR校验**：校验范围包括起始位、功能码、数据长度和数据块的所有字节，逐字节异或，结果为2字节
4. **温度偏移**：所有温度数据均有 +40 偏移，实际温度 = 传输值 - 40
5. **参数范围**：设置保护参数时，参数值必须在有效范围内，否则BMS将返回设置失败
6. **参数保存**：修改保护参数后会自动保存到Flash，重启后生效
7. **字节序**：多字节数据字段采用小端格式（低字节在前）
8. **BLE 与编码**：手机端须对 FFF1 开启 Notify 后再收数，向 FFF2 写指令发数；文本字段编码见第 3.1 节（GB18030）
